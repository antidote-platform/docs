## What Antidote does

Antidote is a two-part safety platform for AI systems, delivered as a
managed cloud service. Use whichever side (or both) fits your stack.

<Columns cols={2}>
  <Card title="Data Integrity" icon="shield-check" href="/data-integrity/user-guide">
    Scan training datasets for label errors, outliers, poisoning, bias,
    leaked secrets, and prompt-injection payloads. Triage, heal, and
    audit the results from a single workspace.
  </Card>
  <Card title="Runtime Security" icon="bolt-lightning" href="/runtime-security/guide">
    A drop-in firewall for LLM traffic. Scan prompts before they reach
    the model and responses before they reach the user. Catches prompt
    injection, PII / secret leakage, and dangerous tool calls.
  </Card>
</Columns>

The two surfaces share the same scanners, the same event store, the
same admin console, and the same workspace. They ship independently so
you can adopt one without committing to the other.

## How the surfaces relate

|                      | Data Integrity                                          | Runtime Security                                             |
| -------------------- | ------------------------------------------------------- | ------------------------------------------------------------ |
| **What it inspects** | Datasets at rest (images, text, NIfTI, detection)       | Live LLM prompts and responses                               |
| **When it runs**     | On upload, on schedule, on demand                       | On every request, inline                                     |
| **Integration**      | Web UI, REST API, scheduled scans                       | Reverse proxy (swap `base_url`) or REST scan endpoints       |
| **Latency budget**   | Minutes. Scans are batch jobs on shared workers.        | Milliseconds. Sits in the chat hot path.                     |
| **Output**           | Issue lists, healed datasets, compliance reports        | `allow` / `redact` / `block` verdicts, events, drift charts  |

If you only need one side today, start there. Wiring up the other
later is additive: the workspace, API keys, and observability
surfaces already know about both.
